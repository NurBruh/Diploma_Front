# 🎯 Инструкция: Как работает отслеживание изменений

## Как это работает?

### 1. **Первый запуск**
- При первом запуске приложение загружает данные с сервера
- Эти данные сохраняются как "базовые" в localStorage браузера
- История изменений пуста

### 2. **Изменение данных на сервере (mokky.dev)**
Когда вы изменяете данные студента в mokky.dev:
- Зайдите на https://mokky.dev/
- Найдите свой проект
- Измените данные студента (например, фамилию, курс, счет и т.д.)
- Сохраните изменения

### 3. **Обнаружение изменений**
- Нажмите кнопку **"Актуализировать"** (зеленая в шапке)
- Приложение загрузит новые данные с сервера
- Автоматически сравнит новые данные с предыдущими
- Если есть отличия - создаст запись в истории изменений
- История сохранится в localStorage

### 4. **Просмотр истории**
- Нажмите кнопку **"История"** у любого студента
- Откроется модальное окно
- Вы увидите:
  - Все изменения для ЭТОГО конкретного студента
  - Дату и время каждого изменения
  - Что изменилось (старое → новое значение)
  - Кто внес изменение

## 📋 Пошаговый тест

### Шаг 1: Очистите историю
```
1. Нажмите "Очистить историю" (красная кнопка)
2. Подтвердите действие
3. Нажмите "Актуализировать"
```
Теперь текущие данные сохранены как базовые.

### Шаг 2: Измените данные студента на mokky.dev
```
1. Откройте mokky.dev
2. Найдите студента (например, ID=1)
3. Измените фамилию: "Иванов" → "Петров"
4. Измените курс: 3 → 4
5. Сохраните
```

### Шаг 3: Обновите данные в приложении
```
1. Вернитесь в приложение
2. Нажмите "Актуализировать"
3. Данные обновятся
```

### Шаг 4: Проверьте историю
```
1. Найдите этого студента в таблице
2. Нажмите кнопку "История"
3. Вы увидите:
   - Фамилия: Иванов → Петров
   - Курс: 3 → 4
   - Дата и время изменения
```

## 💾 Где хранятся данные?

### localStorage браузера:
1. **studentChangeHistory** - история всех изменений
2. **previousStudentData** - предыдущая версия данных для сравнения

### Структура:
```javascript
// История изменений
{
  "1": [  // ID студента
    {
      "id": 1699876543210,
      "date": "09.11.2025, 15:30:45",
      "editor": "Менеджер ОР",
      "changes": [
        {
          "field": "Фамилия",
          "oldValue": "Иванов",
          "newValue": "Петров"
        }
      ]
    }
  ]
}
```

## 🔄 Как работает сравнение?

```javascript
При нажатии "Актуализировать":
1. Загружаем новые данные с сервера
2. Для каждого студента:
   - Берем старые данные из localStorage
   - Сравниваем с новыми данными
   - Если есть отличия → создаем запись в истории
3. Сохраняем новые данные как "предыдущие"
4. Обновляем историю в localStorage
```

## ⚠️ Важно!

### История уникальна для каждого студента
- Каждый студент имеет свою историю
- История хранится по ID студента
- Изменения одного студента не влияют на других

### История сохраняется в браузере
- История хранится в localStorage
- При очистке кэша браузера история удалится
- Каждый браузер имеет свою историю

### Ограничения
- Хранится последние 10 изменений для каждого студента
- История не синхронизируется между устройствами
- Для полноценной системы нужен backend

## 🛠 Команды

### Актуализировать (зеленая кнопка)
- Загружает свежие данные с сервера
- Сравнивает с предыдущими данными
- Создает записи об изменениях
- Обновляет таблицу

### Очистить историю (красная кнопка)
- Удаляет всю историю изменений
- Удаляет сохраненные предыдущие данные
- Следующая актуализация сохранит текущие данные как базовые

### История (синяя кнопка у студента)
- Показывает историю конкретного студента
- Отображает все изменения с визуализацией
- Показывает дату и автора изменений

## 🎨 Визуальные подсказки

- 🔴 **Красный с зачеркиванием** = старое значение
- 🟢 **Зеленый** = новое значение
- ➡️ **Стрелка** = направление изменения
- 🕐 **Серый текст** = дата и время
- 👤 **Синий бейдж** = автор изменения
